# M2 phase 1 — showcase replays

## Purpose

Milestone 2 needs **every corpus spell visible individually** — including the 20 fusions, which have
never been rendered. `showcase` casts ONE given spell in a controlled scene and exports a **normal
replay** (schemaVersion "1"): the existing Unreal ReplayPlayer plays showcase files with **zero
changes**. The law stands: the sim produces every event; nothing synthesizes events outside it.

## Input shapes (auto-detected)

`showcase <spellfile>` accepts, by shape — a JSON object that:

1. carries a `"spell"` key → **seed card** (`fixtures/seeds/*.seed.json`) or **fusion record**
   (`arena/fusions/*.record.json`): the embedded proposal spell is extracted;
2. otherwise → a **bare proposal spell** (id/tier/cast/delivery/clauses at top level).

Every input is compiled through the existing strict gate (`SpellJson.Parse` + `SpellCompiler.Compile`,
default tables) — an invalid spell fails loudly, it is never patched.

## Scene definition

- **Caster** at (0,0,0), named after the spell id; **inert target** `dummy` at (8,0,0) — standard
  arena spacing (`FightEngine.SpawnDistance`). The dummy **never acts** (zero casts, by test).
- Both entities' HP = **10 × TierBudget(spell tier)** so every effect resolves without an early
  kill; caster mana = **10 000** (ample — cost never gates the one cast).
- The caster casts the spell **exactly once at t=0**: `self` delivery targets the caster,
  `groundAoE` aims at the dummy's position, everything else targets the dummy.

## End conditions

After the cast, the sim ticks on the standard 0.5s quantum until **quiescence**: sim-time has passed
the end of every effect the events promised — `max(t + duration)` over non-resisted `StatusApplied`,
`ShieldGranted`, `ZoneSpawned`, and `StatModified` events (recomputed each quantum, so a zone that
applies statuses extends the horizon) — **plus a 1s tail**, capped at **30s** sim time
(`--max-seconds` overrides the cap; the batch uses the default). `endReason` records which end fired:
`quiescent` or `cap`.

## Replay compatibility (hard requirement)

Showcase files are exported through the **existing replay writer** — same header shape
(`kitA` = spell id, `kitB` = `"dummy"`), same event schema, `schemaVersion` stays `"1"`. Showcase
metadata rides in **additional** header keys (`mode: "showcase"`, `maxSeconds`), which every schema-1
reader (Unreal included) ignores. `replay-verify` accepts showcase files: strict header validation
admits `quiescent|cap` as end reasons **when `mode` is `showcase`**, and `--diff-against-live`
re-runs the showcase (same spell file, seed, cap) instead of a fight and diffs byte-for-byte.

## Batch layout

`showcase-batch` runs `showcase` (seed 1, default cap) over **all 6 seed cards and all 20 fusion
records** → `arena/showcases/<id>.replay.json`, where `<id>` is the input's **file stem**
(`ember`, `steam`, `murk-7ff0b332`, …) — the fusion stems already carry the parents-hash for the two
rediscovered duplicates, so names are collision-proof by construction. It also writes
`arena/showcases/manifest.json`: per spell — `id`, display name (seed name / concept name), element,
tags, clause verbs (recursive), delivery type, band values used (per band family), and the replay
file. **The manifest doubles as the G2 coverage input for the Unreal grammar work.**

## Vocabulary coverage

From the manifest corpus, `showcase-batch` also generates
`docs/specs/2026-07-11-m2-vocabulary-coverage.md`: every **verb, delivery, element, status, and band
value** (per band family, reflected from `BalanceTables` so the universe cannot drift from the
tables) that occurs in the 26 showcased spells, with the spells that exhibit it — and an explicit
**missing-tokens** section listing every vocabulary token that occurs NOWHERE. Those are grammar-spec
items with no test material: named, not hidden.

`src/Spellcraft/**`, `schema/`, `prompts/**`, `fixtures/*.proposal.json`,

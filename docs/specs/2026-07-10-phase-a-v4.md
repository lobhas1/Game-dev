## The single moving lever

This round makes the **`ifStatus` 'major' amplify multiplier** an experiment **run config**, sweeps
it offline on the *existing v3 kits*, and ‚Äî after a human lock-in ‚Äî confirms on fresh kits.

- **Moving:** the `major` amplify multiplier (committed default `2.5`), supplied per run.
- **Fixed:** `hpScale = 8`, `mana = 250` (both as registered in v3). The `minor` (1.5) and
  `extreme` (4.0) bands are **untouched**.
- **Watch-item (no action):** the sweep's low end reaches `1.5`, where `major` would equal `minor`
  and the `minor < major < extreme` ordering collapses; `extreme = 4.0` stays fixed this round. If
  the registered value lands at/near `minor`, a future round should re-space all three bands
  together. Logged only ‚Äî no change here.

**Re-pricing is free.** The oracle emits *bands*, never numbers; the compiler prices bands into
numbers. So the same committed kits can be re-compiled at any `major` multiplier with **no
re-generation** ‚Äî the closed-vocabulary architecture working as designed.

## Machinery

- **`BalanceOverrides { float? AmplifyMajor }`** (Spellcraft) ‚Äî an optional pricing override
  **threaded explicitly** (no mutable global/static state; parallel tests + determinism forbid it).
  `null` field ‚áí the committed `BalanceTables`. **Null overrides == today's tables, bit-for-bit.**
- **`SpellCompiler.Compile(spell, overrides = null)`** ‚Äî only the `major` amplify band consults the
  override; every other band and magnitude is unchanged. **`BalanceTables` values are not mutated.**
- **`Sim(seed, overrides = null)`** ‚Äî stored and used by Sim's internal compile (uncompiled spells
  cast through a Sim are priced with the override). **`Kit.Load(path, overrides = null)`** applies
  it at load (Arena kits compile eagerly, so this is where the Arena fight's pricing is set) and so
  validates that the override compiles.
- **Arena run config** ‚Äî `--amplify-major f` on `fight` / `tournament` / `sweep`; `FightConfig`
  carries `AmplifyMajor` (`null` ‚áí engine default). `results.csv` gains an **`amplifyMajor`**
  column recording the *effective* multiplier. `evaluate`'s mixed-config key becomes
  **`(hpScale, mana, amplifyMajor)`** and the verdict prints all three; legacy CSVs without the
  column read as `2.5`.
- **`sweep <kits> --seeds A..B --hp-scale 8 --amplify-majors 2.5,2.25,2.0,1.75,1.5`** ‚Äî for each
  value, load the corpus at that multiplier and run the same-tier round-robin; write
  `docs/experiments/2026-07-10-amplify-sweep.md`: one row per value (same-tier median, swing %,
  end-reason mix, per-tier-class medians). **EXPLORATORY ‚Äî no verdict, no PASS/FAIL.** The `2.5`
  **control row must reproduce v3's criterion numbers (13.9s / 76.9%) exactly**; if it does not, the
  plumbing changed behavior and the run stops.

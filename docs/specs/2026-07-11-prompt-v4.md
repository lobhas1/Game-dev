# Prompt v4 — flavor register + element-verb decoupling

## Findings (from the stranger naming session, 2026-07-08, and the v3 fusion corpus)

The naming premise passed (16% nonsense, bar <20%), but two findings are confirmed:

1. **Flavor register is overwrought** — mystic-filler prose ("trying too hard to be mysterious",
   the stranger's words), with heavy reuse of a small lexicon (hide / blind / threshold / border
   observed in-session).
2. **A hard element→verb lock** — fire→damage 100%, water→heal 100% across the v3 corpus. Traced to
   seed orthogonality (each seed is one element + one archetypal verb), oracle priors, and zero
   prompt guidance to the contrary.

v4 fixes both **by prompt only**. Seeds are **FROZEN** — no new anchor spells this round
(base-collection design is a future round). Both fusion-prompt shas change; the v3 fusion corpus is
archived; F1/F2 re-verify live later (~1€).

## Pre-registered v4 criteria (these never move)

- **V1 naming regression:** fresh-stranger kill test on the v4 corpus stays GREEN (<20%
  nonsense). A passed test re-run after changes is a test that can fail; that risk is accepted and
  intended — v1's GREEN does not carry over.
- **V2a banned-lexicon rate:** % of flavor lines containing any frozen banned token ≤10%.
  Banned list (frozen now): threshold, border, veil, whisper(s), essence, ancient,
  forgotten, eternal, realm, shroud, twilight, betwixt, unseen, beyond.
- **V2b repetition:** no single content word appears in >15% of the corpus's flavor lines.
- **V2c blind register preference:** re-fuse the SAME 20 parent pairs under v4; where the name
  converges with v3 (expected often), pair old/new flavor lines; tester picks preferred line,
  blind, shuffled sides, 10 pairs: v4 preferred ≥7/10.
- **V3a** F1 regression ≥90% (was 100%). **V3b** F2 ≥85% AND > decoy (was 95.8% — a small drop is
  the accepted price of decoupling; a large one fails). **V3c** decoupling evidence: ≥2 of the 20
  re-fused concepts express a tag-justified off-stereotype verb for their element (justified = the
  F2 mapper confirms the verb covers one of the concept's own tags).
- **F3-v2** (later, human): 20 trials, decoys SAME-ELEMENT wherever the corpus allows, ≥14/20.

### Measurement definitions (frozen with the criteria)

- Banned-token match: case-insensitive whole word, optional plural `s` (so `whisper(s)`,
  `veils`, `borders` all match).
- Content word (V2b): alphabetic token, length ≥3, not in the frozen stopword list (single source:
  `FlavorEval` in `src/Arena`, mirrored by `scripts/flavor-stats.py`; the banned-list sync test pins
  spec ↔ script ↔ code). V2b counts *lines containing* the word, not occurrences.

## Baseline (v3 corpus, measured BEFORE any prompt edit)

`python scripts/flavor-stats.py arena/fusions` on the 20 v3 fusion records:

| metric | v3 baseline | v4 threshold |
|---|---|---|
| V2a banned-lexicon rate | **5.0%** (1/20 — `unseen` in mudstone-mirror) | ≤10% |
| V2b max content-word repetition | **15.0%** (`fire`, `nothing`, `stone`, `water`, `wind` each in 3/20) | ≤15% |

Caveat: the stranger session's 25 prototype flavor lines were **not transcribed** (the session doc
records only the verdict counts and the observed repeated words: hide, blind, threshold, border), so
the baseline is the record corpus only. The record corpus already sits at the V2b boundary with five
words tied at 15% — the repetition finding is real there too, and V2c (blind preference) is the
criterion that directly tests the register complaint.

## Exact prompt diffs

### `prompts/naming-oracle.md` — add a Register section AFTER the doctrine sentences

> ## Register — how the flavor line must read
>
> The flavor line is at most 14 words, concrete and physical: what the thing does or looks like
> in-world, never how mysterious it is. Prefer specific nouns and verbs over mood abstractions.
> Never use: threshold, border, veil, whisper(s), essence, ancient, forgotten, eternal, realm,
> shroud, twilight, betwixt, unseen, beyond. Vary word choice — do not lean on the same words
> across many spells.

(The six naming-doctrine sentences are byte-identical and stay ABOVE this section; the doctrine sync
test stays green.)

### `prompts/fusion-mechanics-oracle.md` — ONE sentence, appended to "Express the concept"

> An element flavors a spell; it does not dictate its verbs — let the concept's tags decide, and
> off-stereotype combinations (a fire that heals, a water that strikes) are welcome exactly when
> the concept's meaning demands them.

## Machinery

- **`flavor-eval <fusionsDir>`** — prints V2a and V2b against the spec thresholds (PASS/FAIL each);
  the banned and stopword lists live in `FlavorEval` (single source, sync-tested against this spec
  and the Python script).
- **`quiz --same-element-decoys`** — decoys drawn from same-tier peers of the SAME concept element
  wherever the corpus allows; where it can't, falls back to same-tier with a logged note per trial.
- **`evaluate-fusions`** — already prints both prompt shas; now REFUSES a corpus that mixes
  current-sha and stale-sha records (v3/v4 mixing) instead of silently scoring the matching subset.
  Archive first, then evaluate.

## Live runbook (human, later, ~1€ — NOT run in this build)

1. `git mv arena/fusions arena/fusions-v3-archive` (commit the archive).
2. Re-fuse the SAME 20 parent pairs, ONCE each (`fuse-batch` over the v3 pair list; rejections are
   data — never rerun a pair).
3. `evaluate-fusions arena/fusions` (F1/V3a, F2/V3b, quantization) + V3c check (off-stereotype,
   tag-justified verbs — from the F2 mapper's per-fusion coverage).
4. `flavor-eval arena/fusions` (V2a, V2b).
5. `quiz arena/fusions --same-element-decoys --seed 42` (F3-v2 material).
6. Commit with the real numbers in the message.

## Session protocol (one evening, three text verdicts)

- **G1 (V2c register preference):** 10 old/new flavor pairs (name-converged fusions), blind,
  shuffled sides — tester picks the preferred line. v4 needs ≥7/10.
- **G2 (F3-v2):** the 20-trial same-element-decoy quiz, ≥14/20.
- **G3 (V1 naming kill test):** fresh stranger, the HTML prototype, <20% nonsense. **G3 = a
  different tester — never the same person as G1/G2** (the kill test needs someone who has seen
  nothing, and G1/G2 expose flavor lines and mechanics).

# Phase A v3 — health-scaling calibration + protocol machinery

## v1 result

v1 ([docs/experiments/2026-07-07-phase-a-verdict.md](../experiments/2026-07-07-phase-a-verdict.md))
was **OVERALL FAIL**. Generation quality **passed** — C1 first-pass validity 86.7% ≥ 70%. Fight
dynamics **failed** — C2 median duration 1.7s (window 15–90s) and C4 lead-swings 23.3% (≥50%
required); C3 passed. Diagnosed causes: (1) arena HP is a flat 300 while tier budgets grow
×1.6/tier, so one amplified detonation (~1.75–2.0× budget) one-shots any fighter, and same-tier
TTK collapses monotonically by tier; (2) every oracle shield was elemental, and an elemental
shield absorbs only its own element, so in a mixed-element arena defense never functioned
(absorbed=0 across all fights); (3) cross-tier pairings (71% of the corpus) measured the tier
ladder rather than composition. `amplify` (MultiplierBands ×2.5) pricing is implicated but is
**out of scope for v3** and stays a logged future experiment.

## v3 protocol (pre-registered)

- **(a) Criterion corpus = same-tier round-robin only.** Cross-tier pairings become a separate
  informational **ladder** table (per tier-matchup medians); never scored.
- **(b) Each fighter's HP = `hpScale × TierBudget(its own tier)`** — HP tracks the ×1.6/tier budget
  growth instead of a flat 300. **Registered hpScale: 8** — chosen by the human from the committed
  [calibration sweep table](../experiments/2026-07-09-calibration-sweep.md): at scale 8 the same-tier
  median is 29.0s with 61% lead-swings, and the T3 pocket lifts from 1.1s (scale 3) to 15.9s — into
  the 15–90s window. Mana stays 250, logged as a **watch-item**: the out-of-mana share rises with
  hpScale — 38% of same-tier fights end in `oom` at scale 8 (death:80 oom:50 of 130) — so raising
  mana is a candidate future experiment. This registration is the pre-registration moment; the
  number does not move regardless of the verdict.
- **(c) The prompt gains two sentences** (elemental-shield cost; `displace` has no `share`). This
  changes the prompt hash, so v1 `generation.log` rows stop matching under `evaluate` — that is the
  integrity guard **working**, not a regression. v3 confirmation requires fresh kits generated under
  the v3 prompt.
- **(d) The four thresholds are unchanged, verbatim:** first-pass validity ≥70%; median fight
  duration 15–90s sim-time; ≥3 distinct verbs firing per fight; ≥50% of fights show a lead change or
  status-driven swing (sign flip of the HP difference, sampled 1/s).
- **(e) Exploration on the v1 corpus** (the sweep); **confirmation on fresh v3 kits only.**

## Machinery

- **`FightConfig { HpScale = 0, Mana = 250 }`** — `HpScale = 0` means legacy flat 300 HP.
  `FightEngine.Run(a, b, seed, config)`; the seedless-config overload defaults to legacy and
  reproduces today's fights exactly (pinned by a committed golden). HP per fighter =
  `HpScale ≤ 0 ? 300 : HpScale × BalanceTables.TierBudget(kit tier)`.
- **`results.csv` gains trailing columns `hpScale, hpA, hpB, mana`.** `evaluate` tolerates old
  13-column CSVs (legacy config 0 / 250).
- **`tournament <kits> [--same-tier] [--hp-scale k] [--mana N]`** — round-robin, both orderings, all
  seeds; `--same-tier` keeps only equal-tier pairs (the criterion corpus). Over a multi-tier corpus
  it also prints an informational **ladder** table (per tier-matchup median + end-reason mix).
- **`sweep <kits> --seeds A..B --hp-scales 3,4,5,6,8`** — for each scale, run the same-tier
  round-robin and write `docs/experiments/2026-07-09-calibration-sweep.md`: one row per scale with
  same-tier median, swing %, end-reason mix, and per-tier-class medians. Labeled **EXPLORATORY**;
  no verdict, no PASS/FAIL language.
- **`evaluate`** prints the config (`hpScale`, `mana`) it reads from `results.csv` into the verdict
  doc and **ERRORS if rows mix configs**. Scoring is otherwise unchanged: C1 hash-filtered from
  `generation.log`, C2–C4 over the criterion `results.csv` as run.

## Immutable

The four thresholds; `src/Spellcraft/**` (**including `MultiplierBands`/amplify pricing** — a logged
future experiment, untouched here); `schema/`; `fixtures/*.proposal.json`; `tests/Spellcraft.Tests/**`;
and the v1 verdict doc (history is append-only). **With no new flags, every existing command
reproduces today's behavior** — regression proof: `fight` seed 1 → winner ember / death / 37.6s,
pinned by a committed golden projection.

## Deviations (justified)

1. **`hpScale` is an added `results.csv` config-key column** beyond the specified `hpA, hpB, mana`.
   Absolute HP varies by tier within a single valid scaled run, so `hpA/hpB` cannot serve as the
   mixed-config key; `hpScale` can. `hpA, hpB, mana` are present exactly as specified.
2. **The runbook runs the criterion (`--same-tier`) tournament last** before `evaluate` so the
   scored `results.csv` is the criterion corpus; a trailing full-corpus run would clobber it. The
   ladder is read from the full-corpus tournament's printed table. (The task's listed order would
   let the full-corpus run overwrite the criterion results.)
